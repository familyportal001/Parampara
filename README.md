# Parampara — Private Family Tree & Heritage App

> *Parampara* (परम्परा / பரம்பரை / പാരമ്പര്യം) — tradition passed down through generations.

Parampara is a private, invite-only web app for Indian families to build and preserve their family tree, heritage, and memories — together.

---

## What it does

- **Family tree** — add people, relationships, birth and death dates, photos
- **Heritage** — record Gotra, Kula, Vamsham, Kula Daivam and ancestral identity
- **Private by design** — every family is isolated. Members join only by invite link from a family admin. No public profiles, no discovery, no ads.
- **Multilingual** — English, Tamil (தமிழ்), Malayalam (മലയാളം)
- **Works offline** — progressive web app, installable on any device
- **India-first** — built for Indian family structures, naming conventions, and lineage traditions

---

## Why the source code is public

Parampara handles private family data — names, dates, relationships, photos of people across generations. We believe families have the right to inspect exactly how their data is stored and protected.

Making the source code publicly visible is a trust commitment, not an open source licence. You can read every line of how the app works, how Firebase security rules are structured, and how data flows. There are no hidden analytics, no advertising integrations, and no third-party data sharing beyond Firebase (Google) for hosting and authentication.

See our [Privacy Policy](privacy.html) for full details.

## Tech stack

| Layer | Technology |
|---|---|
| Frontend | Vanilla JS, single HTML file, no build step |
| Auth | Firebase Authentication (email + password) |
| Database | Firebase Realtime Database |
| Storage | Firebase Storage (photos) |

| PWA | Service Worker + Web App Manifest |

No frameworks. No npm. No bundler. The entire app is one HTML file — deliberately, so any developer can read and audit it end to end without tooling.

## Security model

- All family data is scoped by a family hash derived from the family key
- Firebase security rules enforce membership before any read or write
- Members join only via single-use invite links (7-day expiry) generated by a family admin
- Pending members cannot read family data until approved by an admin
- Viewer, Contributor, Editor, and Admin roles with granular permissions
- No server-side code — all logic runs in the browser against Firebase directly

The Firebase security rules are in [`firebase-rules.json`](firebase-rules.json) — readable and auditable.

---

## Data & privacy

- Family data is stored in Firebase (Google Cloud, Mumbai region where available)
- No data is sold or shared with third parties
- Families can export all their data as CSV at any time
- Families can permanently delete all their data from the admin panel
- Compliant with India's Digital Personal Data Protection Act 2023
